Still on the board — by priority

  • Backup codes for TOTP — single-use recovery codes for the lost-phone case. Today, lost phone → admin reset (manual). Smallest, highest urgency.
  • QA Jira widgets — the only buildable slice of the QA Hub today (Jira is the only QA data source we have wired). Defect leakage, defect cycle time, regression labels. ~1 day.
  • managerId on the admin user editor — adds managerId to the PATCH schema and surfaces a dropdown in the row editor. Unblocks future Manager Hub work even though the hub itself doesn't exist yet.
  • Manager Hub design conversation — needs cross-user data-access policy (does a manager see raw metrics or anonymized aggregates?) before any code lands.
  • TestRail / Zephyr connector — blocked on QA team picking which tool they use + an API token.
  • Email change flow — currently refused by the profile editor; needs a confirmation-email round-trip + every-session invalidation. Parked.
  • Hard-delete user — soft-delete via status: disabled covers the operational need; hard-delete would orphan audit-log refs. Intentionally not built.
  • Zoho HR sync — populates users.{managerId, zohoEmployeeId, level, hireDate}. Out-of-band integration task; not started.
  • Redis-backed rate limits — in-process limiters are fine on a single instance; cross-instance HA needs Redis. Container is in docker-compose.yml; no code consumes it yet.
  • Production observability stack — pino is structured-JSON-ready; need to wire a log shipper + dashboards + Sentry on the frontend.
