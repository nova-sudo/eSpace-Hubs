02 App overview

A quiet dashboard for loud review seasons.

01
Problem solved

Engineers reconstruct review evidence by hand from 3-4 tools every cycle. By April nobody remembers February — and the strongest receipts get forgotten.

02
Primary users

L1-L3 engineers prepping reviews and 1:1s today. Tech leads scanning 30/90-day pictures of teammates next. Eng managers running portfolio reviews after that.

03
Value

One receipts-ready view across Jira + GitLab + GitHub. AI classifies your goals into trackable widgets. The output is a paste-ready review paragraph — not a spreadsheet.

03 Status · week 2

Two weeks in, most of the spine ships.

SHIPPED Real, in the worktree, built & verified
  • 6-section scroll-snap dashboard (Overview · Review timing · Glance · Goals · Trends · Goal tracking)
  • TTFR / ATTNR / total-idle review-timing engine + drill-down at /reviews
  • AI Analyst — streaming goal classifier with 12-widget catalog
  • Code-rubric grading per PR, cached by (prId, rubricHash)
  • Evidence page → paste-ready paragraph + Markdown / Print-PDF export
  • Weekly snapshots store with compare-two-weeks diff
  • Command palette (⌘K) + global shortcuts (1-6, j/k, g+x chords)
  • Three AI providers (Mistral · GLM · OpenRouter), switchable per session
  • Demo mode — deterministic synthetic dataset (14 PRs, 4 reviewers, 90d)
OUT OF SCOPE · WEEK 2 Deliberately deferred
  • No backend — localStorage-only state across the entire app
  • No team / multi-user views — single engineer protagonist
  • No mobile or tablet UX — desktop-first scroll-snap layout
  • No Zoho Goals integration — role permissions block the API
  • No automated test suite (yet)
  • No write-side actions on integrations — strictly read-only
  • No SSO / org-level auth — per-user PATs and OAuth clients
04 Demo flow

Twelve minutes. Five beats.

01 ~1 min · setup Demo data on

Toggle demo mode before opening the laptop. Synthetic dataset, no tokens needed.

02 ~3 min · headline Review timing

TTFR · ATTNR · idle. Click an idle PR → drill into /reviews. Grade with AI.

03 ~4 min · differentiator AI Analyst

Streaming goal classifier. Watch tokens land. Switch provider mid-flight.

04 ~3 min · payoff Evidence → paragraph

Deterministic draft → AI polish → copy → paste-ready for Workday / Lattice.

05 ~1.5 min · optional Cmd+K + keys

Every action in 3 keystrokes. Fallback if running long: skip and wrap.

09 Architecture · talking points

The bets behind the UI.

Next.js 16 · Turbopack stack App Router · JSX (not TS) for week-2 velocity · Tailwind v4 with CSS variables — the design system is the variables, no theme switcher needed
localStorage-only state key bet Zero infra cost · zero security review · instant offline · custom-event subscriptions + useSyncExternalStore for cross-tile reactivity
OpenAI-compatible AI proxy stack Mistral · GLM · OpenRouter all share /chat/completions shape · adding a 4th provider is a one-line registry entry · keys server-side only
SOLID feature folders stack src/features/{integrations, goal-specs, goal-widgets, analyst, grading, evidence, snapshots…}/ · each owns its store, hooks, components · no cross-feature reaches
Streaming everywhere key bet Classifier streams NDJSON over fetch · UI re-flows per token · no "spinner-then-dump" · failures surface inline, never silently
10 Before vs. after

Six rituals, collapsed.

Before today, every review cycle
  • Open Jira → screenshot tickets → tally manually
  • "Was that PR slow because of me or the reviewer?" — no data
  • Scroll Slack threads to remember PR feedback
  • Re-write the same self-review paragraph every cycle
  • Forget which PRs to highlight by Friday afternoon
  • Spreadsheet across 3 review windows of varying definitions
With Dev Hub today's improvements branch
  • One scroll-snap section, real numbers vs. previous period
  • TTFR vs ATTNR vs idle, ranked by worst PRs
  • /reviews shows every comment with its diff hunk
  • Paste-ready paragraph generated from current metrics
  • "Star" evidence as you go; export bundle with one click
  • "Since last review" preset — same window for every artifact
11 Open questions

Decisions I want from the room.

a. Product scope
  • Stay with the 12-widget catalog or let users define custom kinds?
  • Snapshot auto-capture every Monday, or stay manual?
  • "Stale PR" threshold — currently 7 days; team norms vary
  • Group /reviews by repo or keep flat sorted-by-idle?
b. Technical
  • localStorage → IndexedDB → Postgres-sync — when do we cross which line?
  • Wire Anthropic Claude directly, or stay on OpenRouter for it?
  • Auth model when we go multi-user — SSO via existing GitHub OAuth?
  • CI/CD — Vercel preview environments, or stay local for now?
c. Security & compliance
  • Tokens in browser localStorage — accept the risk, or move to server?
  • SOC 2 / ISO marketing claims — when, by whom?
  • Rotation guidance — surface in onboarding or document separately?
  • Cross-team data when we eventually add team views?
d. Performance & metrics
  • First paint < 1.5s on Wi-Fi — measure properly post-demo
  • AI classifier < 30s for 12-goal tree — currently 15-20s on Mistral
  • Adoption metric: # active engineers per week, post-pilot
  • Self-reported time saved — survey after first review cycle
12 Blockers · contingencies

If something breaks live.

!
Missing AI key

.env.local not in worktree → "no API key" error. Fix: hard-link from main worktree before demo. Verify by clicking Grade once during setup.

!
Network flake

AI provider rate-limits or times out mid-demo. Fall back to deterministic paragraph — it's the headline value anyway. Pre-record the streaming classifier as a 10s MP4 backup.

!
Data scrambled

User accidentally clicks something destructive mid-demo. Fix: ⌘K → "demo off" → "demo on" reseeds deterministically — same numbers every time.

!
Print dialog quirks

Browser print dialog varies by host. Rehearse on the actual demo machine. Keep .md export as primary path; PDF is the bonus.

!
Free-tier OpenRouter

Free model rate-limits during peak. If demoing OpenRouter, queue a paid Mistral fallback in the pill. Switching is one click.

!
Running long

Skip step 5 (Cmd+K). The first 4 carry the story alone. Better to land the wrap-up than rush the keyboard demo.
