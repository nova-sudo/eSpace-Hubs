WK 03 · WHAT COMES AFTER THE DEMO

Five themes. One platform. Built next.

Today's eSpace Dev Hub is a single-engineer tool with localStorage state. Next: org-aware accounts, single sign-on, an HR data spine, a hub family for the rest of the company, and AI that classifies more than just code.

ESPACE.DEV / DEVHUB WK 03 · ROADMAP REVIEW
02 Where we are today

A working demo. Not yet a platform.

01
What ships

Six-section dashboard, AI-classified goals, code-rubric grading per PR, paste-ready evidence paragraphs, weekly snapshot stream, demo mode end-to-end.

02
What it isn't

No accounts. No org tree. No SSO. No team views. No HR data. State is per-browser localStorage. One engineer, one device — that's the demo's edge and its ceiling.

03
What's next

Take the same shape — quiet, opinionated, receipts-first — and lift it into a multi-tenant product across five disciplines. The next five slides each unlock one piece.

03 Five themes

Five things to build. In this order.

01 foundation Admin dashboard

Create employee accounts, assign roles, manage lifecycle. The control plane the rest of the platform depends on.

02 foundation Authentication

Replace per-user PATs with org SSO. Sessions, roles, audit log — the gate every hub sits behind.

03 data spine Zoho integration

Pull employee tree, manager chain, hire dates, levels and review cycles. HR is the source of truth — we read from it.

04 expansion Hub family

DevHub for engineers, plus four siblings: LeadHub, TalentHub, QualityHub, ProductHub. Same shape. Different signals.

05 depth AI evolution

More widget kinds. Better classifier. Citations on grades. Auto-summary writers per role. AI that learns the org's vocabulary.

The first three are sequential — each unlocks the next. Hubs and AI run in parallel once the spine is in.

04 01 / Admin dashboard

Someone has to create the accounts.

Why first Every other theme assumes accounts exist. Without an account model, SSO has nothing to map onto and Zoho has nowhere to land.
Who uses it Org admins (IT / People Ops). Small surface area, high blast radius — locked behind an admin role.
Core jobs Invite by email · assign role & hub access · activate / suspend / archive · org tree import from Zoho · audit log of every state change.
Roles shipped admin · manager · member — extensible per hub (e.g. qa-lead, po-lead).
Out of scope No billing surface. No customer-facing tenant management. Internal-only platform — one tenant, one org.
  • Email invite with one-time link · 7-day expiry
  • Role & hub matrix (a member can sit in DevHub + ProductHub)

  • Every change written to audit log: actor, target, before, after, ts

05 02 / Authentication

Real accounts. Real login.

Today No login. Per-user GitHub / GitLab / Jira PATs pasted into a settings page, stored in localStorage. Works for one engineer on one device.
Next Standard email + password sign-in. Server-side sessions. Roles enforced on every API call. Nothing exotic — just the auth every web app needs.
Sessions HTTP-only · SameSite cookies · 12h sliding expiry. Revocable from the admin dashboard. Sign-out invalidates server-side, not just the cookie.
Passwords Hashed with a modern KDF (argon2id). Reset-by-email flow with one-time tokens. Optional TOTP 2FA in the user's profile settings.
Later SSO (SAML / OIDC) is a v2 option for orgs that want it — same session layer, just an extra IdP step at sign-in. Not required for v1.

06 03 / Zoho integration

HR is the spine. We read from it.

Why now DevHub today doesn't know who reports to whom, when someone joined, or what level they're at. Every smart aggregation (team rollups, peer cohorts, cycle alignment) is gated on this.
Ask of HR A read-scoped Zoho People API client + service account with permission for: people, departments, reporting chain, hire dates, role / level, performance cycles.
What we ingest Employee tree · manager chain · level & grade · hire date · department · review cycle dates · goal taxonomy if maintained.
What we don't No comp. No PII beyond name + work email. No salary, no reviews, no PIP status. Strictly the org-shape and the calendar.
Sync model Nightly delta sync · webhooks for joiners / leavers / role changes. Zoho is source of truth — we never write back without explicit user action.
Data we'd pull · per employeeread-only
Identity work email · full name · employee id (the join key everywhere)
Org shape manager id · department · location · employment type
Career state level / grade · hire date · role title · last promotion date
Cadence performance cycle window · review dates · goal taxonomy if defined
Lifecycle joined · changed manager · left — drives access & team views
What it unlocks
  • LeadHub team rollups — "my reports" auto-populated, no manual roster
  • Peer-cohort comparisons — same level, same hire window, same department
  • Cycle-aware snapshot windows — week 1 of cycle ≠ week 1 of calendar
  • Tenure-aware onboarding — joined this April? Compliance starts April, not Jan 1
Risk · we name it
  • Until HR grants the API role, the entire hub family slips. This is the critical-path ask.
  • Fallback: admin-uploaded CSV for the org tree as v0 — Zoho live in v1

08 04a / LeadHub

Managers need the other view.

DevHub is "my evidence." LeadHub is "my team's evidence" — and the two screens managers actually open every Monday: the rollup, and the delegated-jobs queue.

01
Collect goals across reports

Auto-populated roster from Zoho. One screen shows every report's L1/L2 goals, AI-classification status, and missing-context flags. Nudge a report directly when their goal needs a rubric.

02
Roll up evidence + compliance

Same snapshot stream that powers individual dashboards, aggregated weekly per report. Pass-rate, on-target %, ungraded count — across the team, sortable, filterable.

03
Delegate jobs + track them

"This RFC needs Sara." Assign with a deadline, track via a queue with TTFR / age / completion signals — same engine that grades reviews, applied to delegated tasks.

04
Calibrate without the spreadsheet

At cycle close, all reports' evidence packets surface side-by-side. Same paste-ready paragraph as DevHub — pre-drafted per report — saves the review-week archeology.
